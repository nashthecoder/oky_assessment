Preparation notes — video script bullets

🙋 To the user

  • You tap Edit Profile on iPhone — the app crashes and throws you out completely
  • No warning, no error — just suddenly back at your home screen
  • It happens because your details hadn't fully loaded yet when the screen opened
  • The app reached for your information, found nothing, and didn't know what to do
  • After the fix — it checks first before trying to load anything
  • If your details aren't ready, it waits quietly instead of crashing
  • You stay inside the app — no more random kick-outs

📊 To the stakeholder

  • Girls on iPhone were being crashed out of the app when trying to edit their profile
  • No data was lost — but the experience was broken and trust-eroding for a young audience
  • The root cause: the app assumed user data would always be ready — it wasn't always
  • This fix closes that gap — the screen now handles missing data gracefully
  • Risk reduced: one less crash path on iOS, which is a significant share of Oky users
  • Outstanding: the fix shows a blank screen momentarily — not harmful, but not polished for a 10-year-old user
  • Recommend: follow-up ticket for a loading state and investigation into what triggers the race condition

💻 To the developer

  • Removing the as User cast is the right call — it was hiding a real nullable from TypeScript
  • The useMemo null guard and early return are correct and clean
  • The fallback state in useReducer is dead code — the early return fires first, so it's never reached; remove it or add a comment explaining why it's there
  • Returning null is safe but silent — consider a loading spinner or redirect for this age group
  • The fix treats the symptom — what triggers the null in the first place? Race condition on logout? Worth a follow-up ticket
  • No test coverage mentioned — given this is iOS-specific, a device test against the crash scenario should be confirmed before merge
  • Decision: request changes — replace null render with a loading state, remove dead fallback code; crash fix itself is correct and should go in once those are addressed
